# Game Asset Helper — 웹 UI 사용 가이드

이 문서는 Game Asset Helper(GAH)의 웹 기반 사용자 인터페이스를 처음 사용하는 분을 위한 안내서다. 트레이 앱 부팅부터 에셋 검색, Claude Code와의 협업 워크플로까지 순서대로 설명한다.

---

## 1. 시작하기

### 1.1 트레이 앱 실행

GAH는 윈도우즈 시스템 트레이에 상주하면서 웹 서버를 함께 구동한다.

```powershell
python -m gah --tray
```

정상 실행되면 시스템 트레이 우측 하단에 GAH 아이콘이 나타나고, 기본 브라우저에서 라이브러리 페이지(`http://127.0.0.1:9874/library`)가 자동으로 열린다.

브라우저가 자동으로 열리지 않으면 트레이 아이콘을 우클릭 → **메인 창 열기**를 누르면 된다.

### 1.2 접속 주소

| 상황 | URL |
|---|---|
| 기본 | `http://127.0.0.1:9874` |
| 9874 포트 점유 | `http://127.0.0.1:9875` ~ `http://127.0.0.1:9883` (자동 폴백) |

포트가 충돌할 경우 GAH는 9874부터 최대 9883까지 순서대로 빈 포트를 찾아 사용한다. 모두 점유된 경우 트레이 아이콘을 우클릭 → **종료** 후 재시작한다.

### 1.3 다크 모드 / 라이트 모드

웹 UI는 운영체제의 다크 모드 설정을 자동으로 따른다(`prefers-color-scheme` CSS 미디어 쿼리 사용). Windows 설정 → 개인 설정 → 색 → 기본 앱 색에서 "어둡게"로 바꾸면 GAH UI도 즉시 반영된다.

수동 다크/라이트 토글은 M8(패키징 + i18n)에서 추가될 예정이다.

## 2. 라이브러리 페이지 (`/library`)

(스크린샷: 라이브러리 페이지 전체 화면 — 추후 추가)

라이브러리 페이지는 GAH의 핵심 화면이다. 등록된 모든 에셋을 검색하고, 필터를 적용하고, 상세 정보를 확인할 수 있다.

### 2.1 검색 바

페이지 상단의 검색 입력창에 자연어 쿼리를 입력한다.

- 타이핑 후 **300ms 디바운스** — 입력을 멈추면 자동으로 서버에 검색 요청이 전송된다.
- 검색어 예시: `"짧고 경쾌한 UI 효과음"`, `"pixel art 판타지 캐릭터"`, `"숲 배경 타일셋 초록"`
- 검색어 없이 진입하면 **추가일 기준 최신순**으로 전체 에셋이 표시된다.

검색 바 우측 드롭다운에서 기본 정렬 방식을 선택할 수 있다.

| 정렬 옵션 | 설명 |
|---|---|
| 추가일↓ | 가장 최근에 라이브러리에 추가된 에셋부터 |
| 점수↓ | 하이브리드 검색 점수 높은 것부터 (검색어 입력 시 기본) |
| 이름↑ | 파일명 가나다/알파벳 오름차순 |

### 2.2 ⚙ 고급 — 사이드 패널

(스크린샷: 사이드 패널 열린 상태 — 추후 추가)

검색 바 우측 **⚙ 버튼**을 클릭하면 오른쪽에서 고급 설정 패널이 슬라이드로 열린다. 같은 버튼을 다시 클릭하면 닫힌다.

패널 왼쪽 가장자리에 있는 **리사이즈 핸들**을 클릭-드래그하면 패널 폭을 240~640px 범위에서 자유롭게 조정할 수 있다.

패널 상단에는 **B / C / D** 세 개의 탭이 있다.

---

#### 2.2.1 B 탭 — 필터

라벨, 에셋 종류, 팩 등 다양한 조건으로 결과를 좁힐 수 있다.

**매칭 모드**

| 모드 | 동작 |
|---|---|
| AND | 선택한 모든 라벨을 포함하는 에셋만 표시 |
| OR | 선택한 라벨 중 하나라도 포함하면 표시 |
| NOT | 선택한 라벨을 포함하지 않는 에셋만 표시 |

**라벨 검색**

"🔍 라벨 검색…" 입력창에 키워드를 입력하면 일치하는 axis 칩이 노란 테두리(box-shadow)로 강조된다. 빈 칸으로 되돌리면 강조가 해제된다.

**종류 탭**

`스프라이트 / 시트 / 사운드` 탭을 클릭하면 해당 종류의 에셋에 속한 axis 칩만 표시된다.

**axis 칩**

각 칩은 의미 라벨 axis를 나타낸다(예: `category`, `mood`, `palette`, `sound_category` 등). 칩을 클릭하면 활성(파란/강조색 배경)으로 토글되며 즉시 검색이 재실행된다. 다중 선택이 가능하다. 매칭 모드(AND/OR/NOT)에 따라 선택한 칩들이 결합된다.

**다축 필터 (4개 드롭다운)**

`▶ 팩`, `▶ 벤더`, `▶ 라이선스`, `▶ 상태` 헤더를 클릭하면 상세 필터가 펼쳐진다.

- **팩**: 특정 팩에 속한 에셋만 표시 (다중 선택 가능)
- **벤더**: 특정 벤더의 에셋만 표시
- **라이선스**: MIT, CC0, Asset Store EULA 등 라이선스별 필터
- **상태**: 활성/비활성 팩 포함 여부

---

#### 2.2.2 C 탭 — 표시 옵션

결과 영역의 표시 방식을 제어한다.

| 설정 | 옵션 |
|---|---|
| 보기 | 그리드(카드 열) / 리스트(가로 줄) |
| 카드 크기 | S(소) / M(중) / L(대) |
| 정렬 | 추가일↓ / 점수↓ / 이름↑ |

C 탭의 설정과 검색 바 오른쪽 툴바는 **양방향 동기**된다. 둘 중 어느 쪽에서 바꿔도 나머지가 즉시 반영된다.

**카드 메타 토글**

카드에 표시할 정보를 4개의 체크박스로 조정할 수 있다.

| 체크박스 | 카드에서 표시되는 정보 |
|---|---|
| 라벨 | 상위 라벨 칩 |
| 팩 | 소속 팩 이름 |
| 점수 | 검색 점수 (0~1) |
| 크기 | 파일 크기 |

체크박스를 켜거나 끄면 현재 표시된 카드에 즉시 반영된다.

---

#### 2.2.3 D 탭 — 조정

검색 가중치, 저장된 검색, 통일성 설정을 관리한다.

**프리셋 3가지**

| 프리셋 | 특징 |
|---|---|
| 균형 | 의미 검색 + 통일성 + 피드백 학습을 균형 있게 배분 |
| 통일성 우선 | 이미 채택한 팩·벤더의 에셋을 강하게 선호 |
| 참신성 | 새로운 팩·스타일을 적극적으로 탐색 |

프리셋 버튼을 클릭하면 아래 슬라이더 6개가 자동으로 설정되고 검색이 즉시 재실행된다.

**슬라이더 6개**

`▶ 슬라이더 직접 조정…` 헤더를 클릭하면 세부 조정 영역이 펼쳐진다.

| 슬라이더 | 의미 |
|---|---|
| 의미 | 자연어 임베딩 코사인 유사도 가중치 |
| 키워드 | FTS5 키워드 매칭 가중치 |
| 라벨 | 라벨 점수 매칭 가중치 |
| 통일성 | 프로젝트 사용 이력 기반 일관성 가중치 |
| 신선도 | 최근 추가/분석된 에셋 우선 가중치 |
| 피드백 | 채택/거부 이력 학습 가중치 |

슬라이더를 움직이면 6개의 합이 1.0이 되도록 나머지가 자동 정규화된다. 변경 즉시 Config가 갱신되고 검색이 재실행된다.

> **주의**: 현재 슬라이더 설정은 앱 재시작 후 초기화된다(M8 에서 영속화 예정).

**저장된 검색 CRUD**

자주 쓰는 검색 조건을 저장하고 재사용할 수 있다.

1. 원하는 검색어 + 필터 + 가중치 조합을 설정한다.
2. D 탭 하단 이름 입력창에 저장 이름을 입력 후 **+ 현재 검색 저장** 버튼을 클릭한다.
3. 저장된 검색 목록에 항목이 추가된다.
4. 항목을 클릭하면 저장된 검색어·필터·가중치가 복원되고 검색이 재실행된다.
5. 항목 오른쪽 **×** 버튼을 클릭하면 삭제 확인 후 제거된다.

저장된 검색은 DB에 영속 저장되므로 앱 재시작 후에도 유지된다.

**통일성 요약**

D 탭 하단에서 현재 프로젝트의 팩 사용 분포와 거부된 에셋 수를 간략히 확인할 수 있다. **상세 보기**를 클릭하면 팩별 채택 횟수와 페널티 상세를 모달로 볼 수 있다.

---

### 2.3 결과 영역

(스크린샷: 검색 결과 카드 그리드 — 추후 추가)

검색 결과는 검색 바 아래 카드 형태로 표시된다.

**와이드 카드 (기본)**

각 카드는 왼쪽 60×60 썸네일 + 오른쪽 텍스트 정보로 구성된다.

- 파일명, 소속 팩, 검색 점수, 파일 크기(C 탭 메타 토글에 따라 표시/숨김)
- 상위 라벨 칩 (의미 축 기반 — category, mood, palette 등)
- 사운드 에셋의 경우 ▶ 버튼이 나타난다.

**사운드 인라인 재생**

사운드 카드의 **▶ 버튼**을 클릭하면 카드 아래에 인라인 오디오 플레이어가 나타나 미리 들을 수 있다. 다른 카드의 ▶를 클릭하면 이전 재생은 멈추고 새 파일이 재생된다.

**페이지네이션**

기본 20개 카드가 로드된다. 결과 목록 맨 아래 **더 보기** 버튼을 클릭하면 다음 20개가 추가 로드된다.

---

### 2.4 자산 상세 모달

(스크린샷: 자산 상세 모달 — 추후 추가)

카드를 클릭하거나 카드 포커스 후 **Enter 키**를 누르면 해당 에셋의 상세 정보 모달이 열린다.

**모달 주요 구성**

- 썸네일 (또는 사운드 파형 스펙트로그램)
- 파일명, 팩, 종류, 파일 크기, 해상도(또는 오디오 길이)
- 분석 라벨 전체 목록 (axis별)
- 검색 점수 분해 (의미/키워드/라벨/통일성/신선도/피드백 각각)

**모달 닫기**

- 모달 오른쪽 상단 **×** 버튼 클릭
- **ESC 키** 입력
- 모달 바깥 영역 클릭

**채택 / 거부 버튼**

- **[채택]** — `asset_usage` 테이블에 이력을 기록하고 모달을 닫는다. 채택된 팩·에셋이 이후 통일성 점수에 반영된다.
- **[거부]** — `feedback_records` 테이블에 부정 피드백을 기록한다. 해당 에셋과 팩의 이후 검색 점수가 소폭 내려간다.

---

### 2.5 키보드 단축키 / 접근성

| 키 | 동작 |
|---|---|
| **Tab** / **Shift+Tab** | 검색 입력창 ↔ 결과 영역 포커스 이동 |
| **Enter** (카드 포커스) | 자산 상세 모달 열기 |
| **ESC** (모달 열린 상태) | 모달 닫기 |

카드에는 `role="button"` + `tabindex="0"` 속성이 부여되어 있어 키보드로 탐색 및 선택이 가능하다.

## 3. 팩 페이지 (`/packs`)

(스크린샷: 팩 관리 페이지 — 추후 추가)

상단 네비게이션 바의 **팩 관리** 링크를 클릭하거나 `http://127.0.0.1:9874/packs`로 직접 접속하면 등록된 팩 목록을 카드 그리드로 볼 수 있다.

### 3.1 팩 카드 구성

각 팩 카드에는 다음 정보가 표시된다.

- 팩 이름
- 벤더 / 라이선스
- 에셋 종류 분포 (`sprite:N / sheet:N / sound:N`)
- 전체 에셋 수
- 활성/비활성 토글

### 3.2 팩 활성/비활성 토글

팩 카드 우측 상단 토글을 클릭하면 해당 팩이 즉시 활성화 또는 비활성화된다. 비활성 팩은 카드가 흐릿하게 표시되며, 라이브러리 검색 결과에서 제외된다.

변경 사항은 HTMX fragment 교환으로 페이지 새로고침 없이 반영된다.

## 4. 라벨 관리 페이지 (`/labels/admin`)

상단 네비게이션 바의 **라벨 관리** 링크를 클릭하거나 `http://127.0.0.1:9874/labels/admin`으로 접속한다.

(스크린샷: 라벨 관리 페이지 — 추후 추가)

### 4.1 24 axis 탭

라벨은 의미 축(axis) 24개로 구분된다. 페이지 상단에 탭이 나열되어 있으며, 탭을 클릭하면 해당 axis의 라벨 목록으로 전환된다.

| 그룹 | Axis 목록 |
|---|---|
| 시각 (14개) | category, style, mood, palette, color, view, material, lighting, time_of_day, weather, theme, size_hint, domain, animation |
| 사운드 (10개) | sound_category, sound_mood, sound_timbre, sound_tempo, sound_pitch, sound_loop, sound_environment, sound_instrument, sound_effect_type, sound_duration |

### 4.2 라벨 CRUD

**라벨 추가**

1. 페이지 하단 입력창에 라벨 이름(영문 소문자, 32자 이내)과 설명을 입력한다.
2. **+ 추가** 버튼을 클릭하면 현재 선택된 axis에 라벨이 추가된다.

**라벨 설명 수정**

라벨 행의 설명 텍스트를 더블클릭하면 인라인 편집 모드로 전환된다. 수정 후 **저장**을 클릭하거나 Enter를 누르면 즉시 반영된다.

**라벨 삭제**

라벨 행 오른쪽의 **휴지통(×) 버튼**을 클릭한다. 현재 에셋에서 사용 중인 라벨은 삭제가 차단된다(HTTP 400 응답). 먼저 에셋에서 해당 라벨을 제거해야 삭제할 수 있다.

### 4.3 JSON 내보내기 / 가져오기

**내보내기**

페이지 우측 **JSON 내보내기** 버튼을 클릭하면 현재 등록된 모든 라벨을 `labels_export.json` 파일로 다운로드한다.

파일 형식:

```json
[
  {
    "axis": "category",
    "label": "character",
    "description": "플레이어 또는 NPC 캐릭터",
    "enabled": true,
    "source": "seed"
  },
  ...
]
```

**가져오기**

1. 위 형식과 동일한 JSON 파일을 준비한다.
2. **파일 선택** 버튼으로 파일을 선택 후 **업로드** 버튼을 클릭한다.
3. 이미 존재하는 라벨(`axis` + `label` 조합 기준)은 스킵된다. 새 라벨만 추가된다.

### 4.4 라벨 어휘 변경 알림 (토스트)

라벨이 추가·수정·삭제되면 SSE(`labels_signature_changed` 이벤트)를 통해 브라우저에 알림이 전달된다. 화면 우측 하단에 **"라벨 어휘가 변경됐습니다 — 새로 고침 권장"** 토스트 메시지가 4초간 표시된 후 자동으로 사라진다.

라이브러리 페이지를 열어 둔 채로 라벨을 수정했다면 새로고침(`F5`)해야 axis 칩 목록이 최신화된다.

## 5. Claude 요청 카드 — `request_user_pick` 시나리오

(스크린샷: Claude 요청 카드 — 추후 추가)

Claude Code(MCP 클라이언트)가 `request_user_pick` 도구를 호출하면 GAH 웹 UI에 사용자 선택 요청 카드가 나타난다. 이 기능을 통해 Claude가 에셋 후보를 추려서 사용자에게 최종 선택을 맡기는 협업 워크플로가 구현된다.

### 5.1 카드가 나타나는 방식

Claude Code가 다음과 같이 `request_user_pick`을 호출한다.

```python
# Claude Code MCP 호출 예시 (Claude 가 내부적으로 실행)
await mcp.call_tool("request_user_pick", {
    "candidates": [
        {"asset_id": 42, "reason": "짧고 경쾌함"},
        {"asset_id": 55, "reason": "픽셀 아트 스타일 일치"},
        {"asset_id": 77, "reason": "이 프로젝트에서 3회 채택"}
    ],
    "reason": "UI 클릭 효과음 3가지 중 하나를 선택해주세요",
    "project_id": "my_unity_project",
    "timeout_seconds": 300
})
```

호출 즉시 라이브러리 페이지 결과 영역 **상단에 보라색 strip 카드 그룹**이 나타난다. 헤더에는 🤖 **Claude 요청** 배지와 `reason` 문구가 표시된다.

### 5.2 사용자 선택 방법

| 동작 | 결과 |
|---|---|
| 카드 중 하나의 **[채택]** 버튼 클릭 | MCP에 `picked_asset_id` 반환 + 자동 `record_asset_use(source="claude_pick")` + 카드 그룹 사라짐 |
| 헤더의 **[✕ 거부]** 클릭 | MCP에 오류 코드 `499_user_cancelled` 반환 + 카드 그룹 사라짐 |
| 5분(timeout_seconds) 미응답 | MCP에 오류 코드 `408_timeout` 반환 |

### 5.3 알림 표시

- **헤더 배지** — 대기 중인 pick 요청 수가 헤더 우측에 보라색 숫자 배지로 표시된다. 처리하면 즉시 감소한다.
- **트레이 툴팁** — GAH 트레이 아이콘 위에 마우스를 올리면 `"N건 대기 중"` 메시지가 표시된다.

### 5.4 주의사항

- 브라우저를 새로고침하면 **이미 대기 중인 pick 카드는 다시 나타나지 않는다**. 새 이벤트가 발생할 때만 카드가 추가된다. 단, 타임아웃 내라면 다시 새로고침 없이 Claude가 동일 요청을 재전송하면 카드가 다시 나타난다(v1 한계).
- Claude Code에서 `request_user_pick` 도구 사용법은 [`docs/MCP_USAGE_GUIDE.md`](./MCP_USAGE_GUIDE.md) §5를 참고한다.

1. 위 경로에서 **"어둡게"** 선택

## 9. 문제 해결

### 브라우저가 자동으로 열리지 않아요

트레이 아이콘 우클릭 → **메인 창 열기**를 선택한다. 또는 주소창에 `http://127.0.0.1:9874`를 직접 입력한다.

### 포트 9874에 접속이 안 돼요

GAH가 9874~9883 중 빈 포트를 자동으로 선택했을 수 있다. 트레이 아이콘 위에 마우스를 올리면 현재 사용 중인 포트 번호가 표시된다. 또는 `http://127.0.0.1:9875`, `9876`, ... 순서로 시도해본다.

### Claude 요청 카드가 나타나지 않아요

1. GAH 트레이 앱이 실행 중인지 확인한다(트레이 아이콘 확인).
2. 브라우저가 `http://127.0.0.1:<port>/library` 페이지를 열고 있는지 확인한다.
3. 브라우저 콘솔(F12 → Console)에서 SSE 연결 에러(`EventSource failed`) 여부를 확인한다.
4. 브라우저를 새로고침하거나 트레이 앱을 재시작한다.

### 검색 결과가 비어 있어요

- 에셋 팩이 `%APPDATA%\GameAssetHelper\library\` 아래에 폴더 형태로 추가되어 있는지 확인한다.
- 분석이 아직 진행 중일 수 있다. 라이브러리 페이지 상단 진행률 표시줄을 확인한다.
- B 탭의 필터 조건이 너무 엄격하지 않은지 확인한다. 필터를 모두 초기화하고 빈 검색어로 다시 시도한다.

### 라벨이 삭제되지 않아요

에셋에서 사용 중인 라벨은 삭제가 차단된다. 해당 라벨이 달린 에셋의 분석 결과에서 먼저 라벨을 제거해야 한다. (v1에서는 GUI 직접 삭제가 지원되지 않아 DB 직접 수정이 필요할 수 있다. v2에서 에셋별 라벨 수동 편집 UI 예정.)

### 분석 큐가 멈춰 보여요

- Ollama가 실행 중인지 확인한다. `ollama list` 명령으로 모델 목록을 확인하고, `gemma4:e4b` 모델이 설치되어 있는지 확인한다.
- 로그 파일(`%APPDATA%\GameAssetHelper\logs\gah.log`)에서 에러를 확인한다.

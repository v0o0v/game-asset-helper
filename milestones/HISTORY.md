## Trusted Publishing 패턴 (검증 기록)

- v0.1.0 (M10), v0.1.1, v0.1.2, v0.2.0, **v0.2.2** — 5회 실 publish, 평균 ~30초
- **v0.2.1 은 PyPI 영구 결번** (2026-05-21 발견) — pyproject 가 0.2.0 인 채 tag 만 push → 같은 wheel 빌드 → `skip-existing: true` silent skip ([workflow run](https://github.com/v0o0v/assetcache-mcp/actions/runs/26206683356) 31초 success 로 종료, PyPI 도착 0건). 교훈: `pyproject.toml` + `src/assetcache/__init__.py` **둘 다 bump 한 commit 먼저**, 그 다음 tag.
- `git tag vX.Y.Z; git push origin vX.Y.Z` 한 줄로 자동 publish + GitHub release 생성 (bump commit 전제)
- GitHub Actions workflow: `actions/checkout@v6` + `actions/setup-python@v6` (Node.js 24 호환)
- PyPI CDN lag ~15초 후 `pip install -U assetcache-mcp` 가능
